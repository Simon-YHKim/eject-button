## CLAUDE.md Rule Block

Inserted by `scripts/install-rules.sh`. Marker: `<!-- context-guardian-rules:v1 -->`.

Sections:
1. **작업 범위 제한** — max 5 files/session, one feature at a time, commit before session end
2. **파일 읽기 제한** — block node_modules / .next / dist / .git, no aimless scanning, no full reads of 1000+ line files
3. **작업 요청 방식** — decompose broad requests into small units with user confirmation
4. **컨텍스트 보호** — 80% → generate SESSION_RECOVERY.md, 90% → force new session

Idempotent: re-running skips if marker found.

## SESSION_RECOVERY.md

Generated by `scripts/create-recovery.sh`. Auto-populated fields from git:
- Repo name (`basename $(git rev-parse --show-toplevel)`)
- Current branch (`git rev-parse --abbrev-ref HEAD`)
- Last commit (`git log -1 --oneline`)
- git status / git diff --stat
- Env var names from `.env.example` (values excluded)

Manual fields (user fills after generation):
- 완료된 작업 (checked items)
- 미완료 작업 (unchecked items)
- 핵심 변경 파일 (one-line summaries)

Includes a ready-to-copy "next session prompt" at the bottom.

**Secret scan**: the script aborts if it detects patterns matching `sk-*`, `sk_live_*`, `pk_live_*`, `AKIA*`, `ghp_*`, `xox[bps]-*`. Review and remove before proceeding.

## context_limit_log.json

Managed by `scripts/update-context-log.sh`. Schema:

```json
{
  "model": "claude-opus-4-6",
  "measured_limit": 200000,
  "safety_margin": 0.8,
  "effective_limit": 160000,
  "last_disconnect": "2026-04-14T12:34:56Z",
  "history": [
    { "date": "2026-04-12T10:00:00Z", "measured_limit": 200000, "model": "claude-opus-4-6" }
  ]
}
```

Why empirical (not hardcoded):
- Model updates change the real limit (Claude 4 → 4.5 → 4.6)
- Different subscription tiers have different limits
- Hardcoding would go stale the moment a new model ships

`effective_limit` = `measured_limit × safety_margin`. Default safety_margin = 0.8.

Subcommands:
- `--load` — print current values at session start
- `--record --model X --measured N` — append history + update current
- `--check <tokens>` — warn if current usage is above 80% / 90%
